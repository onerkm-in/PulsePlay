## 3. What behavior changed (visible vs invisible)

### Visible

1. **`/` (root app):** Surface switcher pills no longer expose `BI BI Viz` in text content. Pulse internal header tabs render SVG glyphs instead of emoji + literal "BI" text.
2. **`/settings/ai`:** The page is now structured in 4 conceptual tiers (Assistant → Shared context → Response behavior → Surface-specific behavior). Knowledge pack is no longer in the Assistant tier; it's in Shared context with copy explaining both surfaces use it.
3. **`/settings/setup`:** AI profile field has a "Databricks docs ↗" link next to the label (was inside the tooltip and unreachable). Hint text changes when the profile select is empty for diagnosable reasons.
4. **Provenance footer (AI Insights cards):** reads "Generated by PulsePlay · Source: Default profile · Updated 19 min ago" instead of "AI-generated · Source: default · 19 min ago".
5. **Settings SubSection headings:** sentence case (Current state, Connect and embed, Governance and policy, Mix composition, Status, Policy, Actions) instead of CURRENT STATE / CONNECT AND EMBED / etc.
6. **Tooltip behavior:** opening a tooltip near the edge of a narrow viewport now clamps + flips to stay inside the viewport.

### Invisible

- New `formatProvenanceSourceLabel` helper for human-friendly source mapping.
- New `FieldRow.labelTrailing` prop.
- New `pp-helptip__bubble--align-*` / `--side-*` CSS classes.
- `SubSection` h3 no longer has `text-transform:uppercase` style — purely stylistic, no DOM API change.
