## Top Findings For Claude

1. **P0/P1 Performance: live AI response time misses the product bar.** Rajesh's target is a perceived 5-second response and hard 5-10 second useful output. Observed Ask Pulse was still in `Working out the right query` around 1:22 and only completed after an additional wait; AI Insights status showed about `3:18`. Split this into backend/query latency, polling/status UX, and frontend render/jank. Do not solve by spinner polish alone.

2. **P1/P2 Pulse-ported glyph cleanup is incomplete.** Settings cleanup landed, but AI Insights still renders utility glyphs like `📋`, `↻`, and `</>` in card footers. These should become icon buttons with accessible labels/tooltips, consistent hover states, and no raw emoji/text-glyph feel.

3. **P2 KPI delta glyphs are duplicated.** Rajesh noticed values can show two up arrows or two down arrows in the same delta line, for example a main `▲` plus another `▲` inside the comparison pill/text. The rule should be one physical direction glyph per delta; color/tone carries business meaning. Remove nested duplicate arrows while preserving direction truth and semantic tone.

4. **P1/P2 HelpTips still clip or hide at frame edges.** Rajesh observed edge tooltips going under the frame. This is likely a stacking-context / overflow-clipping issue, not upstream guidance. Do not rely on only increasing `z-index`; if the bubble lives inside a clipped scroll pane, it must render through a top-level portal/floating layer (`document.body` or app overlay root), use fixed coordinates, and run viewport collision detection.

5. **P2 Legacy instructional copy still exists in user-facing setup/settings text.** `playground/src/pulse/settings.ts` still says "AI-generated source/timestamp footer"; `playground/src/pulse/setupStep5.tsx` still explains the footer as `AI-generated | Source: <profile> | <relative-time>`. Rendered output is fixed, but author-facing help copy is not.

6. **P2 Tooltip formatting needs product-quality microcopy.** The HelpTip behavior is fixed, but the bubble reads like a dense paragraph. Use a short title and 2-3 scannable lines. This is frontend `HelpTip` content/CSS, not Genie guidance or prompt configuration.

7. **P2 Mobile/cross-platform remains not proven in this visible desktop pass.** Prior known gaps around dock/pop-out/mobile clamp should remain queued. The final acceptance pass needs explicit narrow viewport screenshots for root, Settings Setup, Ask Pulse, AI Insights, and floating panel.

8. **P2 Typography is not fully synchronized across sections.** AI Insights mixes Pulse-ported CSS variables, inline `fontFamily` styles, mono/code styling, and section-specific styles. Create a single typography contract for AI output: one body family, one heading family, one mono family, and tokenized size/weight/line-height. A formatting pane is useful, but it should write theme tokens/presets rather than letting every section choose its own ad hoc font.

9. **P1/P2 SQL affordance should never open a dead explanatory panel.** The implementation behavior is expected: some narrative sections reuse a sibling stage query instead of running a new warehouse query. But if clicking `</>` repeatedly shows `This section reuses data from an earlier query`, the affordance has low value and feels broken. Acceptance rule: a SQL action must either (a) show the section's own SQL, (b) show the reused source SQL in-place labelled `Reused from <section>`, (c) jump/open the reused source SQL directly, or (d) be hidden/disabled with a short tooltip when no traceable SQL exists. Do not keep a large empty-state explanation as the primary outcome.

10. **P3 Test warnings still exist.** Full tests pass, but Vitest logs React style-shorthand warnings in `FirstRunWizard`, ECharts 0x0 jsdom warnings, and an `act(...)` warning in viewport integration tests. Not blockers, but they make regression output noisy.
