## New / Remaining Findings

### P1 - `AI brain` still appears in visible AI Settings copy

Route: `/settings/ai`

Visible text still includes: `The AI brain that answers your questions.`

This contradicts the naming audit recommendation to use `AI assistant` or `Assistant profile`. It is not structurally dangerous, but it keeps the enterprise copy feeling playful/internal.

### P1 - Special-character cleanup is incomplete outside the root surface switcher

The root surface switcher is clean, but Settings still exposes decorative/action glyph text in visible UI:

- `ŌÜÖ Settings`
- `ŌåÉ Back to app`
- `ŌÜĪ Test proxy`
- `ŌÜĪ Test selected profile`
- `Databricks docs ŌåŚ`
- `Full embed form ŌåÆ`
- `Tune Insights behavior ŌåÆ`
- `Layout & display ŌåÆ`
- `Proxy & diagnostics ŌåÆ`
- `Advanced / reset ŌåÆ`
- `Browse Knowledge Base ŌåÆ`

Some of these can remain as icon affordances, but they should not be text characters in labels if the goal is fully polished/human-friendly UI.

### P2 - Legacy `AI-generated` helper remains in source, apparently unused

`playground/src/pulse/visual.tsx` still contains an unused `renderInsightsProvenance()` function that returns:

```text
AI-generated
Source: default
```

`rg` found no call sites, and the active section footer uses the new humanized copy. Still, leaving the old helper around is a future regression trap and causes source audits to keep flagging `AI-generated`.

### P2 - Tooltip click state can leave multiple `role="tooltip"` nodes open

While opening multiple HelpTips in sequence, the currently visible tooltip stayed unclipped, but an earlier tooltip node remained in the DOM offscreen with `role="tooltip"`.

This was not visible to the user, but it may matter for accessibility tooling. Recommended follow-up: make HelpTip mutually exclusive globally, or close on document pointer/focus movement.

### Existing Deferred Bugs Still Reproduced / Still Deferred

- AI profile select is empty even though proxy reports profiles. The improved hint makes the state understandable, but the data-layer bug remains.
- Power BI secure-link shape vs adapter token shape remains outside this pass.

## Verdict

Claude's pass successfully fixed the major `BI BI Viz` regression and delivered the shared-assistant IA direction. The Settings AI parent/child flow is much clearer now.

Do not call the naming/copy cleanup fully complete yet. A focused follow-up should remove the remaining `AI brain` wording, convert visible arrow/lightning/gear glyph text into icon components or cleaner verbs, and remove the stale `renderInsightsProvenance()` helper.
