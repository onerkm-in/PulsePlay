The problem

AI is trapped inside each BI tool

  • Power BI's AI only sees Power BI; Tableau's only Tableau.

  • A different "ask the data" experience per tool.

Why the timing is right

  • We already own Databricks (Genie, Unity Catalog, Mosaic models & agents) + cloud LLMs.
  • Those investments are under-leveraged — no single surface points them at what users look at.
  • A thin layer turns existing spend into one governed, plain-language experience — no new licences.

Q&A
Cross-examination cheat sheet

If they ask… you say

“Why not just Power BI Copilot?”
It only works in Power BI with Microsoft's model. We run multiple tools and want our own approved brain, governed centrally.

“Are you building an LLM/agent?”
No — we orchestrate Genie, Foundation Models, agents we already own. PulsePlay is the thin experience + governance layer.

“Can we trust the AI for decisions?”
Grounded in governed data, every answer trust-tagged by our validator. Structure enforced today; quality scoring roadmapped. Decision-support with explicit trust.

“Do credentials/data leak?”
Credentials never touch the browser; the proxy mints scoped tokens. Sensitive fields masked before any model. UC is the boundary.

“What does it cost?”
No licence — internally built. A small support team + only the compute/tokens we use on platforms we already pay for.

“Is it production-ready?”
Architecture & backbone tested; Power BI + Genie is the hardening focus. Internal v0.x — solid base, phased rollout.
